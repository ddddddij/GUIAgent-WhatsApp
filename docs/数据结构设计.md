# 数据结构设计

---

## 背景说明
本项目是一个**复刻 WhatsApp 的 Android 应用（基于 MVVM 架构,参考“docs/MVVM架构要求.md”）**，主要用于 **GUI Agent 自动化任务执行与验证**。
你需要设计的数据结构，**仅用于支持任务执行状态的记录与验证**，而不是完整业务系统。

---

## 参考项目结构
请严格参考如下项目结构风格：
app/src/main/java/com/example/whatsapp_sim/
├── domain/
│ ├── model/ # 数据模型（你需要设计的内容）
│ └── repository/ # 数据仓库


app/src/main/
├── assets/
│ └── data/ # JSON数据源（需要配合模型）

## 设计约束
请务必遵守以下规则：
### 严格任务驱动
- 只允许为「任务清单中的行为」设计数据结构
- 不允许扩展额外功能（如推荐系统、真实视频/通话等）

### 面向“验证信号”，而不是完整业务
你的数据结构能够验证任务是否完成

---

### 必须可被 GUI Agent 检测

每个模型必须具备：
- 明确的行为记录字段（如 timestamp / id / 状态）
- 可用于断言的数据（如列表、状态位）
---

### 对齐项目设计风格

你必须遵循：
#### domain/model 层
- Kotlin data class
- 单一职责（一个文件一个核心模型）

#### assets/data 层
- JSON 文件必须与 model 对应
- 用于模拟数据源


## 你的任务
1. 设计一下我需要的数据结构
2. 你的设计要完全基于我的任务清单，不在任务清单的任务，永远不需要设计对应的数据结构
3. 严格按照参考项目结构来创建文件
4. 只需要写对应的数据结构的.KT文件,暂时不要写检验任务是否完成的逻辑
5. 目前仅需设计数据结构，暂时不用设计初始数据
